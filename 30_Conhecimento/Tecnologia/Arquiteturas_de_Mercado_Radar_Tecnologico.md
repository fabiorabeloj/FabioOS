# Arquiteturas de Mercado - Radar Tecnologico

## Objetivo

Esta nota **nao arquiva anuncios**.

Ela documenta arquiteturas, padroes, ferramentas e tendencias identificadas em demonstracoes, propagandas, conferencias, artigos, videos e materiais tecnicos.

Todo conteudo deve ser convertido em conhecimento reutilizavel.

## Processo de extracao

Para cada imagem, video ou demonstracao, identificar:

### 1. Problema

Qual problema esta sendo resolvido?

### 2. Arquitetura

Como o sistema funciona?

Fluxo:

```text
Usuario
  -> ...
  -> Resultado
```

### 3. Tecnologias

- linguagens;
- frameworks;
- bancos de dados;
- APIs;
- infraestrutura;
- cloud;
- IA;
- ferramentas.

### 4. Conceitos

Quais conceitos aparecem?

Exemplos:

- MCP;
- RAG;
- agentes;
- Context Engineering;
- Event Driven;
- Spec-Driven Development;
- Multi-Agent Systems;
- Vector Database;
- observabilidade.

### 5. Padroes recorrentes

O que aparece repetidamente em empresas diferentes?

### 6. Aplicacao no FabioOS

Como essa arquitetura pode ser incorporada?

- imediatamente;
- futuramente;
- descartar.

### 7. Nivel de prioridade

| Nivel | Significado |
|---|---|
| 5/5 | Essencial |
| 4/5 | Muito importante |
| 3/5 | Importante |
| 2/5 | Opcional |
| 1/5 | Apenas referencia |

## Principio

Nenhum material e arquivado por ser propaganda.

Ele so entra no conhecimento quando revela uma arquitetura, um padrao recorrente ou uma solucao tecnica aplicavel ao FabioOS.

## Uso no FabioOS

Este radar deve ser usado antes de:

- comprar ou assinar nova ferramenta;
- instalar agente novo;
- criar subfase nova;
- alterar roadmap;
- promover piloto para producao;
- expor dados a servico externo.
