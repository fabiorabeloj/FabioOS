# Leis do PRIMUS

## Funcao

Consolidar as leis operacionais do PRIMUS em uma pagina consultavel. Esta nota e uma sintese; a fonte completa permanece no Google Doc.

## Leis Fundamentais

| Lei | Regra |
|---|---|
| Prisma triangular | PRIMUS opera por tres faces inseparaveis: Enciclopedia Funcional, Instancia Controlada e Persistencia. |
| Circuito fechado | Toda operacao valida segue E -> I -> P. |
| Canon central | D&D 5e e o centro de regra quando houver conflito mecanico. |
| Precedencia | Canon > Setting > Homebrew > Framework. |
| Framework limitado | Framework nao substitui regra; so altera tema, topologia, entidades e estetica permitida. |
| Tipagem universal | Nada entra no sistema sem Type. |
| Regra de encaixe | Toda entrada declara onde entra, o que pode afetar e o que nunca pode afetar. |
| Rastreabilidade minima | Toda verdade do sistema aponta para fonte, pagina/trecho ou decisao manual justificada. |
| Pipeline editorial | Conteudo deve passar por captura, materializacao, integracao e instanciamento. |
| Simetria local | Cada familia tem template proprio; nao existe template universal para tudo. |
| Instanciamento | Entrada relevante precisa dizer como vira jogo. |
| Contrato de missao | Missao sem objetivo, gate, sucesso/falha, risco, recompensa e DeltaP nao e missao PRIMUS. |
| Portabilidade | Uma entrada pode ser portable, persistent ou local. |
| Persistencia registravel | Consequencia vira DeltaP explicito. |
| Retorno ao hub | O ciclo jogavel retorna a um ponto de controle, como a Cantina. |
| Validacao transversal | E, I e P devem ser validados separadamente. |

## Leis Evolutivas do Documento

- Missao 0001 nao deve iniciar antes da arquitetura essencial.
- Canon resolve primeiro; PRIMUS so cobre lacunas nao resolvidas pelo canon.
- Tempo, economia e recursos magicos nao devem ser reinventados sem necessidade.
- Estado depende do tipo do objeto.
- Relacoes surgem do jogo, de DeltaP e de persistencia.
- Incompletude pode ser produtiva, mas ignorancia estrutural bloqueia progresso.
- Persistencia precisa de escopo: local, campanha, linha de mundo ou canon derivado.
- Player View e camada formal: ver nao significa possuir.

## Criterios de Aceite

Uma nova entrada PRIMUS so pode ser promovida quando:

- possui Type;
- possui fonte ou justificativa;
- informa Box/Livro/Subbox;
- declara `Affects` e `NeverAffects`;
- possui pelo menos um uso jogavel ou motivo claro para permanecer apenas enciclopedica;
- nao viola precedencia Canon > Setting > Homebrew > Framework.
